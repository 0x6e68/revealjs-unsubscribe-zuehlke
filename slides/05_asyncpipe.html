Magic "| async"

What is "| async"?

  • Angular pipe operator: unwraps a value from an asynchronous primitive
  • When the component gets destroyed, the async pipe unsubscribes automatically to avoid potential memory leaks
  • Comes "out of the box" with Angular

Demo: clean up - | async

Summary

  • No (un-)subscribe needed, async pipe does all the work
  • Best solution for simple "get & show" cases
  • Can be combined with elvis operators to render components only when some data is present

Summary - (Dis-)advantages

  • (+) No (un-)subscribe needed
  • (+) Really easy to use / understand
  • (+) Can also used by Promises
  • (-) Only useful for simple use cases
